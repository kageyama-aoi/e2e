# `tree_generator.py` リファクタリング概要

このドキュメントは、プロジェクトのディレクトリ構成を視覚化するツール `tree_generator.py` の設計と機能についてまとめたものです。

## 1. 目的

1.  **構成図の自動化**: 手動で README の構成図を更新する手間を省き、常に最新の状態を保ちます。
2.  **設定の集中管理**: 除外リストやマーカー設定を `AppConfig` クラスに集約し、保守性を高めています。
3.  **可読性の向上**: 日本語の Docstring と適切なコメントにより、ツールの動作を容易に理解できるようにしています。

## 2. 主要機能

### 2.1. 設定クラス (`AppConfig`)
ファイルの冒頭で定義されており、以下の設定を簡単にカスタマイズできます。
- `DEFAULT_EXCLUDE_DIRS`: ツリーから除外するディレクトリ（`node_modules`, `.git` 等）。
- `MARKER_START / MARKER_END`: README.md 内の更新範囲を特定するタグ。

### 2.2. README.md 自動更新モード (`-u` オプション)
`README.md` 内の以下のマーカーで囲まれた範囲を自動的に最新のツリー図で置換します。
```markdown
<!-- TREE_START -->
(ここが自動更新されます)
<!-- TREE_END -->
```

### 2.3. フィルタリングとソート
- 隠しファイル（`.` で始まるもの）の表示切替。
- ディレクトリを先に表示し、その後にファイルをアルファベット順で表示する自然なソート機能。

## 3. 使い方

```bash
# README.md を最新の構成図で更新する
python tree_generator.py -u

# ツリー図を新しいファイルとして書き出す
python tree_generator.py -o project_tree.md
```

このツールにより、プロジェクトの構成変更が即座にドキュメントに反映されるようになります。
