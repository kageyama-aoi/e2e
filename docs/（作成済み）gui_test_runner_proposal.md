### 提案：Tkinterを使用したGUIテストランナー

現在の `Run_Shimamura_Syokai_Test.bat` を、より直感的に操作できるPython/Tkinter製のGUIアプリケーションに置き換えることを提案します。

#### 目的
- コマンドライン操作に不慣れなユーザーでも、簡単にE2Eテストを実行できるようにする。
- プロファイル名の入力ミスを防ぎ、操作性を向上させる。

#### 主な機能
1.  **プロファイルの自動検出**:
    - `env/` ディレクトリ内にある `.env.shimamura.*` という形式のファイルを自動的にスキャンし、利用可能なプロファイルを検出します。
2.  **GUIによるプロファイル選択**:
    - 検出されたプロファイルの一覧をラジオボタンとしてGUIに表示し、ユーザーが選択できるようにします。
3.  **テスト実行**:
    - 「テスト実行」ボタンをクリックすると、選択されたプロファイルで `syokai_touroku.js` テストが実行されます。
4.  **(オプション) リアルタイムログ表示**:
    - テスト実行中のコンソールの出力をGUI上のテキストエリアに表示し、進捗をリアルタイムで確認できるようにします。

#### 実装方針
- **言語**: Python 3
- **GUIライブラリ**: Tkinter (Python標準ライブラリ)
- **ファイル名**: `run_shimamura_test_gui.py` (プロジェクトルートに配置)

#### 処理フロー
1.  アプリケーションを起動すると、`env/` ディレクトリからプロファイルリストを自動で読み込みます。
2.  読み込んだプロファイル名を元に、GUIウィンドウ内にラジオボタンを動的に生成します。
3.  ユーザーがプロファイルを選択し、「テスト実行」ボタンを押します。
4.  選択されたプロファイル名を使って、内部的に以下のコマンドを組み立てて実行します。
    ```shell
    npx codeceptjs run ./tests/shimamura/syokai_touroku.js --profile [選択されたプロファイル]
    ```
5.  テストの実行が完了すると、その旨をメッセージで通知します。

#### 期待される効果
- **操作性の向上**: GUIによる直感的な操作で、誰でも簡単にテストを実行できます。
- **ヒューマンエラーの削減**: プロファイル名を手入力する必要がなくなり、タイプミスがなくなります。
- **メンテナンス性の向上**: 新しいプロファイル（`.env`ファイル）を追加した際に、コードを修正することなく自動で選択肢に反映されます。
